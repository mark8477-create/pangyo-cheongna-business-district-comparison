# 시스템 요구사항 구현 가능성 점검

점검 기준: `final_exam.html` §4  
점검일: 2026-06-13

## 최종 판정

필수 기능 4종과 정적 GitHub Pages 기술 조건을 모두 구현할 수 있는 분석 데이터는
준비되었다. 그러나 현재 `web/`에는 데이터만 있고 HTML·CSS·JavaScript가 없으므로,
시스템 요구사항을 이미 충족한 상태는 아니다.

## 필수 기능 점검

| 요구사항 | 구현 가능성 | 준비된 입력 | 구현 시 주의사항 |
|---|---|---|---|
| 두 구역 경계와 필지·건축물 컬러맵 | 가능 | 경계, 필지, 건축물, 용도지역 GeoJSON | 지역 전환 또는 나란히 보기 UI 구현 필요 |
| 30·60분 등시간권과 수치 | 가능 | 4개 등시간권 폴리곤과 인구·가구·종사자 속성 | 30·60분 전환 버튼 또는 슬라이더 필요 |
| 비교 통계표와 차트 | 가능 | 요약, 용도지역, 건축물, 산업, 누적 접근성 JSON | 산업 코드의 한글 명칭 매핑 필요 |
| 필지·건축물 클릭 속성 | 가능 | 팝업용 속성을 포함한 필지·건축물 GeoJSON | 건축물 연면적·용적률, 필지 지목·미건축 여부 표시 |

## 기술 조건 점검

| 조건 | 현재 상태 | 판정 |
|---|---|---|
| 정적 사이트 | 모든 시스템 데이터가 GeoJSON·JSON | 가능 |
| 별도 서버·DB 없이 동작 | 정적 파일 총량 약 4.6MB | 가능 |
| 베이스맵 | 아직 미선정 | OSM 또는 VWorld 선택 필요 |
| 전처리 스크립트 포함 | `scripts/08_rebuild_comparison_metrics.py` 등 존재 | 충족 가능 |
| README 출처·기준시점·처리 과정 | 최신 기준 반영 완료 | 준비 완료 |
| GitHub Pages 배포 | Git 저장소와 Pages 설정 없음 | 미완료 |

## 현재 차단 요소

1. `web/index.html`, 스타일시트, JavaScript 애플리케이션이 없다.
2. 현재 작업 폴더는 Git 저장소가 아니며 GitHub Pages 배포 설정도 없다.
3. 산업 대분류 데이터가 `cp_bnu_003`, `cp_bem_003` 형태의 SGIS 코드이므로
   차트에서 사용할 한글 산업명 매핑이 필요하다.

## 성능 및 구현 권장사항

- `web/data/` 전체는 약 4.6MB로 정적 배포에 무리가 없다.
- 누적 등시간권 GeoJSON은 약 3.1MB이므로 초기 화면에서는 불러오지 않고,
  교통 탭 진입 또는 시간 슬라이더 활성화 시 지연 로딩한다.
- 30·60분 등시간권 파일은 약 0.84MB이므로 초기 지도 레이어로 사용 가능하다.
- 필지와 건축물 GeoJSON은 모두 유효하며 클릭 팝업 구현에 필요한 속성을 포함한다.
- 한 화면 비교는 두 개의 동기화 지도 또는 단일 지도 지역 전환 방식 모두 가능하다.

## 구현 완료 판정 조건

- 두 지역 경계·필지·건축물 또는 용도지역 컬러맵이 정상 표출된다.
- 30분·60분 등시간권을 전환할 수 있고 인구·종사자 수치가 함께 표시된다.
- 토지이용·교통·인구사회 비교표와 차트가 한 화면에서 비교된다.
- 필지·건축물 클릭 시 속성 팝업이 열린다.
- GitHub Pages URL에서 정적 사이트가 정상 동작한다.
