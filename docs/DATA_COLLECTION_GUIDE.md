# 데이터 수집 가이드

## 1. 가장 먼저 수집할 데이터

경계가 확정되어야 모든 자료를 같은 공간 범위로 자를 수 있습니다.

### 제1판교테크노밸리 경계

- 상태: 수집 및 공간 검증 완료
- 기준: 「성남판교지구 택지개발사업 도시지원시설 지구단위계획 시행지침」
- 포함 대상: 도시1~5, 주12~15·주29~30, 위3~4
- 주의: 사용자가 최초 제시한 `위1~3`이 아니라 2019년 1월 28일 시행지침 원문에
  명시된 `위3~4`를 기준으로 사용함
- 원천자료:
  - `data_raw/boundary/pangyo/prom_pangyo_candidate_pnu_2026-06-12.xlsx`
  - `data_raw/boundary/pangyo/prom_pangyo_spatial_2026-06-12/`
  - `data_raw/boundary/pangyo/pangyo_urban_support_guideline_2019-01-28.pdf`
- 검증 결과:
  - 후보 PNU 152개 모두 PROM 토지대장 polygon에서 확인
  - 누락 PNU 0개, 초과 PNU 0개
  - 주차장용지 6필지, 주유소용지 2필지 확인
  - 최종 경계 면적 `674,923.28㎡`
  - 시행지침 지정개요 면적 `663,575㎡` 대비 오차율 `1.7102%`
- 처리 원칙: 구역 밖까지 이어지는 도로·공원 필지 전체를 병합하지 않고, 검증된
  필지 polygon을 PROM 직접 디지타이징 구역계로 잘라 병합함
- 최종 산출물:
  `data_processed/boundaries/pangyo_validation/pangyo_main_boundary.geojson`

### 청라국제업무지구 경계

- 상태: 수집 및 GeoJSON 생성 완료
- 설정 근거: IFEZ에서 국제업무지구로 제시한 블록을 PROM 지도에서 직접 디지타이징
- 포함 블록: `B1, B2, B9, B10, C1, C2, M5, M6`
- 원천자료:
  - `data_raw/boundary/cheongna/prom_cheongna_ibd_spatial_2026-06-12.zip`
  - `data_raw/boundary/cheongna/prom_cheongna_ibd_pnu_2026-06-12.xlsx`
- 검증 결과:
  - 디지타이징 구역계: 유효한 단일 Polygon
  - 경계와 교차하는 PNU 32개 확인
  - 경계 면적 `359,301.46㎡`
- 현재 GeoJSON:
  `data_raw/boundary/cheongna/cheongna_ibd_boundary.geojson`

PDF나 이미지 도면만 구할 수 있다면 원본 도면과 함께 기준점 또는 좌표가 보이는
자료도 넣어야 지오리퍼런싱할 수 있습니다.

## 2. 토지이용 데이터

- 출처 후보: VWorld, 토지이음, 판교·IFEZ 개발계획 공간자료
- 지역: 판교와 청라 각각
- 필요한 필드: 용도지역/용도지구명, 세부 용도, 면적, 고유 식별자
- 선호 형식: SHP, GeoPackage, GeoJSON
- 차선 형식: 속성표가 포함된 CSV와 폴리곤 파일
- 투입 위치: `data_raw/landuse/`

두 지역에서 같은 분류체계를 쓸 수 있도록 최소한 `업무`, `상업`, `주거`,
`교육·연구`, `기타`로 재분류할 수 있어야 합니다.

## 3. 건축물 데이터

- 출처 후보: 건축HUB, VWorld 건물 공간정보, 건축물대장
- 지역: 판교와 청라 각각
- 필수 필드: 주소 또는 좌표, 주용도, 연면적, 대지면적
- 권장 필드: 층수, 건축면적, 사용승인일, 건물 식별자
- 선호 형식: 건물 폴리곤과 속성 CSV 또는 GeoPackage
- 투입 위치: `data_raw/building/`

주용도별 연면적 구성비, 평균 용적률, 미개발 필지 비율, 개발시점 분포를
산출할 예정입니다. 건축물대장 CSV와 건물 폴리곤이 따로 있다면 연결 가능한
관리번호가 있는지 확인해야 합니다.

## 4. SGIS 데이터

- 현재 상태: 인구·사업체·종사자 통계 및 집계구 공간경계 수집 완료, 가구 자료 대기
- 출처: SGIS
- 공간단위: 집계구
- 지역: 판교·청라 경계와 60분 접근권을 포괄하는 수도권 범위
- 필수 자료: 집계구 경계, 인구, 가구, 사업체 수, 종사자 수, 업종 분류
- 기준연도: 가능한 한 모든 항목을 동일 연도로 통일
- 투입 위치: `data_raw/sgis/`

현재 확보 자료:

- 서울특별시(`11`): 집계구 코드 19,097개
- 인천광역시(`23`): 집계구 코드 6,362개
- 경기도(`31`): 집계구 코드 28,394개
- 인천 서구(`23080`): 집계구 코드 1,281개
- 성남시 분당구(`31023`): 집계구 코드 926개
- 2023년 인구총괄
- 2023년 사업체·종사자 대분류 및 총괄사업체 통계
- 2025년 2분기 집계구 공간경계(`2025-06-30`, EPSG:5179)
- 저장 위치: `data_raw/sgis/request_1781257203152/`
- 공간경계 위치: `data_raw/sgis/boundaries/`

주의:

- 인구·사업체·종사자는 2023년으로 기준연도를 통일함
- 공간경계는 2025년 2분기 자료로 통계와 기준시점이 다름
- 대분류 종사자·사업체의 `N/A`는 SGIS 비밀보호 또는 미제공 값으로 0 처리함
- 별도 총괄종사자 파일이 없어 대분류 종사자수를 집계구별 합산함

업무지구 및 접근권과 겹치는 집계구 값은 겹침 면적 비율로 배분합니다. 따라서
통계표만이 아니라 해당 연도의 집계구 경계 파일도 반드시 필요합니다.

`scripts/04_preprocess_sgis_by_pnu_boundary.py`로 PNU 기반 표준 경계·필지
GeoJSON, SGIS 통계 표준화 CSV, 공간중첩 및 면적비례 배분 결과를 생성했습니다.

## 5. 지하철 네트워크

- 상태: 수집 및 무결성 점검 완료
- 출처: LMS 제공 `subway_network`
- 분석 파일: `network/nodes.tsv`, `network/links.tsv`
- 출발역: 판교역, 청라국제도시역
- 필요한 링크 속성: 역 연결 관계와 소요시간(초)
- 투입 위치: `data_raw/subway_network/`
- 점검 결과:
  - 역 노드 915개
  - 링크 1,192개
  - 끊어진 노드 참조 0개
  - 판교역 경강선·신분당선 노드 확인
  - 청라국제도시역 공항철도 노드 확인
  - 데이터 기준일 `2026-05-05`, 2032년 계획노선까지 포함

기본 분석은 `begin`과 `effective_begin`을 이용해 동일 기준일에 운영 중인
네트워크만 사용합니다. 계획 또는 미개통 노선은 기본 결과에 포함하지 않습니다.

## 6. OSM 도로망

- 상태: 수집 및 업무지구 경계 내부 지표 산출 완료
- 출처: OpenStreetMap
- 범위: 각 업무지구 경계의 1km 버퍼에서 다운로드 후 원 경계 내부로 clip
- 필요한 요소: 운전 가능한 도로 링크와 노드
- 투입 위치: `data_raw/osm/`
- 산출 지표: 도로밀도, 교차로밀도, 필요 시 인근 IC 직선거리
- 수집 코드: `scripts/06_preprocess_osm.py`
- 다운로드일: 2026-06-12
- 라이선스·출처 표기: `© OpenStreetMap contributors`, ODbL
- 결과: 판교 도로밀도 `7.833 km/km²`, 교차로밀도 `20.759개/km²`
- 결과: 청라 도로밀도 `0.682 km/km²`, 교차로밀도 `0개/km²`

## 7. 수집 순서와 완료 기준

| 순서 | 데이터 | 현재 상태 | 완료 기준 |
|---:|---|---|---|
| 1 | 두 대상지 경계 | 완료 | 검증된 외곽 Polygon과 처리 기록 확보 |
| 2 | 지하철 네트워크 | 완료 | `nodes.tsv`, `links.tsv` 무결성 확인 |
| 3 | SGIS | 통계·공간경계 확보, 가구 대기 | 집계구 공간경계와 통계표 연결 및 면적비례 배분 완료 |
| 4 | 토지이용 | 원본 일부 확보, 전처리 대기 | 두 지역을 동일 분류로 재분류 가능 |
| 5 | 건축물 | PROM 원본 확보, 전처리 대기 | 주용도·연면적·대지면적 연결 가능 |
| 6 | OSM | 완료 | 도로 링크·노드 및 경계 내부 밀도 지표 산출 완료 |

## 8. 사용자가 파일을 넣은 뒤 확인할 사항

각 파일마다 `data_raw/source_inventory.csv`에 다음을 기록합니다.

- `status`: `collected`
- `source`와 `source_url`
- 실제 다운로드 날짜와 데이터 기준일
- 파일명, 형식, 좌표계
- 이용 조건 또는 라이선스
- 누락 필드나 주의사항

현재 경계와 지하철 네트워크 수집은 완료되었고, SGIS 인구·사업체·종사자 통계도
확보되었고, 집계구 공간경계 면적비례 배분과 OSM 도로망 분석도 완료되었습니다.

현재 구현된 경계 검증 스크립트:

- `scripts/01_make_pangyo_boundary_from_prom.py`
- `scripts/01_verify_pangyo_boundary.py`
- `scripts/04_preprocess_sgis_by_pnu_boundary.py`

판교 경계 결과의 필지별 전체면적, 경계 포함면적, 포함비율 및 검토 플래그는
`data_processed/boundaries/pangyo_validation/pangyo_main_pnu_verified.csv`에
기록되어 있습니다.
