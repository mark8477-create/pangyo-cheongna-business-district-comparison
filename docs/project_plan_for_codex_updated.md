# 스마트시티 이론과 실제 기말 프로젝트 구현 계획서 for Codex

## 0. 프로젝트 방향성 요약

과제명은 **「데이터로 진단하는 업무지구의 성공과 실패 — 판교테크노밸리 비교분석 시스템 구축」**이다.

본 프로젝트는 다음을 목표로 한다.

1. 성공 사례인 **제1판교테크노밸리**와 활성화가 저조한 업무지구인 **청라국제업무지구**를 비교한다.
2. 공공데이터 기반 비교분석 시스템을 구축한다.
3. 토지이용, 교통망, 인구사회 지표를 정량적으로 비교한다.
4. 업무지구 성공요인을 데이터 근거와 함께 도출한다.
5. 분석 시스템은 GitHub Pages로 배포하고, 보고서의 핵심 수치와 그림은 시스템에서 확인 가능해야 한다.

평가의 중심은 화려한 UI가 아니라 **분석 내용, 비교 설계의 엄밀성, 논리적 전개, 시스템과 보고서의 연계성**이다.

## 1. 최종 비교 대상

### 1.1 기준 지역

- **제1판교테크노밸리**

제2판교는 포함하지 않는다.  
이유는 제1판교가 판교테크노밸리의 핵심 성공 사례이며, 비교 대상인 청라국제업무지구와 기능적 비교를 하기 위해 분석 범위를 명확하게 제한하기 위함이다.

### 1.2 비교 지역

- **청라국제업무지구**

청라국제도시 전체가 아니라, IFEZ에서 국제업무지구로 제시한 다음 블록을 기준으로 한다.

```text
B1, B2, B9, B10, C1, C2, M5, M6
```

### 1.3 청라 경계 설정 근거

IFEZ 자료에서 국제업무지구의 위치가 다음과 같이 제시되어 있다.

```text
청라국제도시 B1, B2, B9, B10, C1, C2, M5, M6 일원
```

따라서 본 프로젝트에서는 해당 블록들을 청라국제업무지구의 핵심 분석 경계로 설정한다.

“일원”이라는 표현은 해당 블록 중심의 사업 범위를 의미하므로, 임의로 청라국제도시 전체를 포함하지 않는다.  
GIS 분석에서는 가구 및 획지 결정도에서 해당 블록들이 차지하는 실제 공간 범위를 디지타이징하여 하나의 Polygon으로 병합한다.

최종 생성 파일:

```text
data_raw/boundary/cheongna/cheongna_ibd_boundary.geojson
```

## 2. 핵심 연구 질문

## Q1. 왜 판교테크노밸리는 기업과 일자리가 집중되었는가?

검증 방향:

- 광역 대중교통 접근성
- 업무시설 및 연구시설 집적도
- 사업체 및 종사자 규모
- 산업구조

## Q2. 왜 청라국제업무지구는 계획 대비 활성화가 저조했는가?

검증 방향:

- 실제 개발 실현 정도
- 업무시설 연면적 및 주용도 구성
- 종사자 밀도 및 사업체 밀도
- 수도권 노동시장 접근성

## Q3. 업무지구 성공을 결정하는 핵심 요인은 무엇인가?

최종 보고서에서는 최소 3가지 성공요인을 도출한다.

예상 성공요인:

1. 광역 노동시장 접근성
2. 업무·연구 기능의 집적도
3. 산업생태계 규모와 종사자 기반
4. 개발 실현도
5. 주변 교통·도로망 조건

## 2.1 보고서의 핵심 논증 구조

보고서는 지표를 병렬적으로 나열하지 않고 다음 인과 경로를 검증하는 구조로 작성한다.

```text
계획된 토지이용이 실제 건축물과 도로로 실현됨
→ 기업과 일자리가 집적될 수 있는 물리적 기반 형성
→ 종사자·사업체와 산업생태계가 형성됨
→ 광역 노동시장 접근성이 집적 효과를 강화함
→ 업무지구 활성화 수준의 차이로 나타남
```

이 경로는 인과관계를 확정하는 것이 아니라, 판교와 청라의 차이를 설명하는
데이터 기반 가설이다. 보고서 결론에서는 상관관계와 인과관계를 구분한다.

### 가설별 검증 설계

| 가설 | 핵심 지표 | 해석 기준 |
|---|---|---|
| H1. 개발 실현도가 높을수록 업무기능이 형성된다 | 건축물 수, 연면적, 평균 용적률, 미건축 필지 비율, 도로밀도 | 계획 용지의 실제 물리적 개발 수준 비교 |
| H2. 업무·연구시설 집적이 고용 생태계 형성으로 이어진다 | 주용도별 연면적, LUM, 종사자·사업체 밀도, 산업 대분류 구성 | 단순 개발 여부를 넘어 업무기능의 질과 집적 비교 |
| H3. 광역 노동시장 접근성이 업무지구 경쟁력을 강화한다 | 30·60분 도달 가능 역·인구·종사자, 누적 접근성 곡선 | 동일 시간 안에 접근 가능한 노동시장 규모 비교 |
| H4. 내부 연결성이 개발 실현 및 활동을 보조한다 | 도로망 밀도, 교차로 밀도 | 구역 내부 이동·필지 접근 조건 비교 |

분석 순서는 `개발 실현도 → 기능·고용 집적 → 광역 접근성 → 종합 성공요인`으로
고정한다. 이를 통해 “무엇이 다른가”에서 끝나지 않고 “그 차이가 왜 중요한가”를
설명한다.

```text
청라국제도시역
```

- 청라국제도시의 대표 광역철도 접근역
- 청라국제업무지구와 연결되는 핵심 대중교통 거점
- 수도권 지하철 네트워크 분석에 포함 가능한 운영 역

주의:

- 향후 계획역 또는 미개통 노선은 본 분석의 기본 기준으로 사용하지 않는다.
- 제공된 subway_network.zip에서 begin/effective_begin 컬럼을 활용할 수 있으나, 기본 분석은 동일 기준연도의 운영 네트워크를 사용한다.
- 수집된 네트워크는 역 노드 915개, 링크 1,192개이며 끊어진 노드 참조는 없다.
- 데이터 기준일은 `2026-05-05`이고 2032년 계획노선까지 포함하므로 운영시점 필터링을 반드시 적용한다.

## 4. 과제 필수 데이터 반영 계획

과제에서 반드시 사용하도록 제시한 데이터는 다음 4종이다.

| 필수 데이터 | 출처 | 본 프로젝트 활용 |
|---|---|---|
| 집계구 단위 인구·종사자수 | SGIS | 인구사회 분석, 등시간권 도달가능 인구·종사자 산출 |
| 토지이용계획 + 건축물대장 | PROM, VWorld, 건축HUB | 용도지역 구성비, 건축물 주용도 구성비, 용적률, 개발 실현도 |
| 도로망 | OpenStreetMap | 도로망 밀도, 교차로 밀도, 도보 접근성 보조지표 |
| 수도권 지하철 네트워크 그래프 | LMS 제공 subway_network.zip | Dijkstra 기반 30분·60분 등시간권 분석 |

모든 분석은 두 지역에 대해 **동일한 기준연도, 동일한 산출 방식, 동일한 공간처리 방식**을 적용한다.

## 6. 토지이용 분석 계획

## 6.1 필수 지표

### 1. 용도지역 구성비

데이터:

공간단위:

|---|---:|---:|---:|
| 판교 | | | |
| 청라 | | | |

---

### 2. 건축물 주용도 구성비

데이터:

- 건축HUB 또는 세움터 건축물대장
- 건축물 단위 주용도, 연면적, 대지면적, 층수, 사용승인일, 용적률

공간단위:

- 원본: 건축물
- 집계: 업무지구 경계 내부 건축물

산출 방식:

```text
주용도별 연면적 합계 / 전체 연면적
```

주요 분류:

```text
업무시설
교육연구시설
근린생활시설
판매시설
문화 및 집회시설
주거시설
기타
```

---

### 3. 토지이용 혼합도 LUM

지표:

```text
LUM = -Σ(pi ln pi) / ln(n)
```

여기서:

- pi = 각 용도별 연면적 또는 면적 비율
- n = 용도 유형 수

해석:

- 0에 가까움: 단일 용도 집중
- 1에 가까움: 여러 용도가 균형 있게 혼합

주의:

- 판교와 청라 모두 동일한 용도 분류 체계를 사용한다.
- 용도 분류를 과도하게 세분하지 않고 업무, 상업, 주거, 교육연구, 기타 정도로 재분류한다.

---

### 4. 개발 실현 정도

산출 후보:

1. 평균 용적률

```text
총 연면적 / 총 대지면적
```

2. 미개발 필지 비율

```text
건축물이 없는 필지 면적 / 전체 필지 면적
```

3. 사용승인일 기반 개발 시점 분포

```text
연도별 사용승인 건축물 수 또는 연면적
```

보고서에서는 청라가 실제로 어느 정도 개발되었는지 확인하기 위해 개발 실현도 지표를 핵심적으로 사용한다.

## 7. 교통망 분석 계획

과제 3-2의 핵심은 **업무지구 중심부의 대중교통 접근성**이다.

## 7.1 핵심 분석: 등시간권

### 입력 데이터

- subway_network.zip
  - nodes.tsv
  - links.tsv

### 분석 방법

1. nodes.tsv에서 판교역, 청라국제도시역 node를 찾는다.
2. 분석 기준일에 운영 중인 노드·링크만 `begin`과 `effective_begin`으로 필터링한다.
3. `timeFT`, `timeTF`를 방향별 edge weight로 사용하고 환승 링크를 포함한다.
4. 판교역의 경강선·신분당선 노드는 모두 출발 후보로 두고, 각 역명까지의 최소
   도달시간을 사용한다. 청라국제도시역도 동일 원칙을 적용한다.
5. NetworkX Dijkstra 알고리즘으로 모든 역까지 환승 포함 최단시간을 계산한다.
6. `0~60분`을 5분 단위 누적 구간으로 나누고 30분·60분 결과를 필수 산출물로 저장한다.
7. 각 시간 구간의 도달 가능 역에 동일한 `800m` 보행권 버퍼를 생성한다.
8. 같은 시간 구간의 역 버퍼를 먼저 병합하여 환승역·인접역의 중복 집계를 방지한다.
9. 병합한 역세권 Polygon을 해당 시간의 지하철 기반 등시간권으로 정의한다.
10. 등시간권과 SGIS 집계구를 intersection하고 교차면적 비율로 인구·가구·종사자·
    사업체를 배분한다.

```text
접근권 배분값 =
집계구 통계값 × (집계구와 병합 역세권의 교차면적 / 집계구 전체면적)
```

### 등시간권 공간단위 결정

- 과제에서 지정한 원자료 단위를 유지하기 위해 격자로 변환하지 않고 집계구를 사용한다.
- 교수자가 요구한 핵심역 출발 기준을 적용하므로 업무지구 경계에서 핵심역까지의
  접근시간은 기본 등시간권에 포함하지 않는다. 이 차이는 분석 한계로 명시한다.
- 800m 버퍼는 도착역에서 최종 목적지까지의 보행 가능 범위를 근사한다.
- 결과 명칭은 연속적인 실제 통행시간 표면이 아니라
  `환승 포함 지하철 도달역 기반 800m 역세권 접근권`으로 명시한다.
- 800m 선택의 민감도를 확인할 필요가 있으면 500m 또는 1km 결과를 보조 분석으로
  비교하되, 판교와 청라에는 동일한 반경을 적용한다.

### 도달역에 800m 버퍼를 적용하는 이유

`subway_network`는 역과 역 사이의 운행시간 및 환승 링크 시간을 제공하지만,
도달한 역이 실제로 영향을 미치는 주변 공간 범위는 제공하지 않는다. 역 좌표만
사용하면 도달 가능한 공간이 면적이 없는 점으로 표현되어, SGIS 집계구의 인구·
가구·종사자와 공간결합할 수 없다. 따라서 도달역을 주변 생활권과 노동시장으로
변환하기 위한 공간화 규칙이 필요하다.

본 분석은 각 도달역 주변 `800m`를 보행으로 접근 가능한 대표 역세권으로 가정한다.
보행속도를 약 `80m/분`으로 보면 800m는 약 10분 보행거리와 대응하며, 역 주변의
잠재적 주거·고용 인구를 포괄하면서 지나치게 넓은 지역을 포함하지 않는 절충 범위다.

```text
시간 t의 접근권 =
환승 포함 지하철 최단시간이 t분 이하인 모든 역의 800m 버퍼를 병합한 영역
```

같은 시간대의 버퍼는 먼저 하나로 병합한다. 이를 통해 인접역 또는 환승역의
역세권이 겹칠 때 동일 집계구 인구·종사자가 중복 계산되는 문제를 방지한다.
병합 영역과 SGIS 집계구의 교차면적 비율을 이용해 도달가능 인구·가구·종사자·
사업체를 배분한다.

이 방식의 800m는 `subway_network`에 포함된 이동시간이 아니며, 지하철 도달역을
공간 범위로 표현하기 위한 별도 역세권 가정이다. 따라서 결과는 엄밀한
`총 이동시간 t분 이내 보행 포함 등시간권`이 아니라 다음과 같이 해석한다.

> 핵심역에서 환승을 포함해 지하철로 t분 이내 도달 가능한 역들의 800m 역세권에
> 포함되는 잠재 인구·종사자 규모

고정 800m를 적용하므로 지하철 이동시간이 t분에 가까운 역에서도 약 10분의 추가
보행 범위를 포함할 수 있다. 이 한계는 보고서에 명시하고, 두 지역에 동일한
반경과 산출법을 적용하여 비교 가능성을 확보한다.

## 8.3 SGIS 면적비례 배분 방식

집계구와 업무지구 경계가 일치하지 않으므로 다음 방식을 사용한다.

```text
가중치 = 집계구와 업무지구가 겹치는 면적 / 집계구 전체 면적
```

예시:

```text
어떤 집계구의 종사자 수 = 1,000명
업무지구와 겹치는 면적 비율 = 0.35

업무지구 배분 종사자 수 = 1,000 × 0.35 = 350명
```

주의:

- 인구·가구·종사자·사업체 모두 동일 방식 적용
- 면적비례 배분의 한계는 보고서의 분석 한계에서 명시
- 건축물이 밀집된 업무지구의 경우 면적비례 배분이 실제 분포와 다를 수 있음을 언급
- 접근권 분석에서는 같은 시간 구간의 역세권을 먼저 병합한 뒤 집계하여 중복 계산 방지
- 2023년 통계와 2025년 2분기 공간경계의 집계구 코드 연결률을 검증하고 기준시점 차이를 명시

## 10.4 성공요인 도출

세 영역 결과를 종합하여 최소 3가지 성공요인을 제시한다.

도출 구조:

1. 성공요인의 이름을 명확히 제시한다.
2. 토지이용·교통·인구사회 중 최소 두 영역의 수치를 연결한다.
3. 판교와 청라의 정량적 격차를 제시한다.
4. 해당 요인이 왜 업무지구 활성화와 관련되는지 해석한다.
5. 관찰자료만으로 인과를 확정할 수 없다는 한계를 덧붙인다.

예상 결론 후보:

1. 개발계획의 실제 실현이 업무기능 형성의 선행조건이다.
2. 업무·연구시설과 기업·종사자의 집적이 산업생태계를 형성한다.
3. 환승을 포함한 광역 노동시장 접근성이 집적 효과를 강화한다.

모든 성공요인은 시스템에 표출되는 수치와 연결한다.

## 14. Codex 구현 지시 요약

Codex는 다음 방향으로 구현한다.

1. Python 기반 전처리 스크립트를 작성한다.

4. 웹 시스템은 web 폴더에서 정적 파일로 구현한다.
5. GitHub Pages에서 바로 동작하도록 서버 의존성을 제거한다.
6. 모든 지도와 차트는 전처리된 데이터를 불러와 표시한다.
7. README에는 데이터 출처, 기준연도, 처리 과정, 실행 방법을 기록한다.
8. 보고서에서 사용할 수 있도록 summary JSON을 별도로 생성한다.

    금융보험업, 전문과학기술업 등 핵심 산업 중심으로 비교
  - 산업 대분류 `N/A`는 0 처리하고 총괄값과의 차이를 별도 기록

- [ ] **건축물 중복 제거 기준**
  - 권장안: `bd_mng_id` 기준으로 중복 제거
  - 일반건축물·집합건축물 기록 중복 여부 확인 후 연면적 집계

- [ ] **업무지구 내부 건축물 포함 기준**
  - 권장안: 건축물 geometry가 업무지구 경계와 교차하면 해당 건축물 전체를 포함
  - 경계에 일부만 걸친 건축물을 면적비례 처리할지 여부 결정

- [ ] **미건축 필지 정의**
  - 권장안: 분석 대상 필지 중 건축물대장 geometry와 하나도 교차하지 않는 필지
  - 미건축 필지 수 비율과 면적 비율을 모두 산출할지 결정

- [ ] **토지이용 혼합도 LUM 기준**
  - 권장안: 건축물 주용도별 연면적 비율로 계산
  - 건축물이 없는 지역은 `산출 불가`로 표시하고 임의로 0을 부여하지 않음

- [ ] **SGIS 비밀보호·미제공 값 처리**
  - 현재 처리안: 대분류 `N/A`를 0으로 변환
  - 산업구성 비율 분모는 관측 가능한 대분류 합계를 사용
  - 총괄값과 대분류 합계 차이를 비밀보호·미제공 규모로 기록

### 15.2 현재 합의된 분석 기준

- 집계구를 격자로 변환하지 않고 원 공간단위로 사용
- 통계 기준연도는 2023년으로 통일
- 집계구 공간경계는 확보 가능한 2025년 2분기 자료 사용
- 집계구 코드 연결률 검증 후 면적비례 배분
- 지하철 환승 링크와 방향별 이동시간을 포함한 Dijkstra 최단시간 사용
- `0~60분`을 5분 단위 누적 접근성으로 산출
- 도달역 주변 고정 `800m` 버퍼를 병합하여 역세권 접근권 생성
- 인접역·환승역 버퍼는 먼저 병합하여 인구·종사자 중복 계산 방지
- 판교와 청라에 동일한 방법과 기준 적용

### 15.3 분석 기준 확정 후 시스템 구상 순서

1. 최종 지표와 출력 파일 목록 확정

3. 시스템 화면 구성과 사용자 이동 흐름 설계
4. 지도 레이어, 통계 패널, 클릭 팝업 속성 확정
5. 정적 웹용 GeoJSON·JSON 경량화 기준 확정
6. 전처리 구현 후 시스템 개발 시작

## 16. 청라 업무중심지구 경계 재정의 및 재산출 (2026-06-13)

### 16.1 최종 경계 정의

- 기존 청라 국제업무단지 단독 경계를 대체하여, 국제업무단지 예정지뿐 아니라
  국제금융단지 예정지, 개발계획도상 업무용지, 실제 업무시설 블록을 포함하는
  **청라 업무중심지구**로 분석 경계를 재정의한다.
- 주변 공동주택 중심 주거블록은 제외한다.
- 새 경계는 `청라국제업무지구_소재지_prom.xlsx`의 PNU 117개와 PROM 필지
  geometry를 연결하고 병합하여 생성한다.
- 최종 경계 파일은 `data/processed/cheongna_boundary_final.geojson`과
  `public/data/cheongna_boundary_final.geojson`으로 관리한다.

### 16.2 자료 기준시점

- PROM 필지·건축물 공간자료는 현재 확보된 **2026년 PROM 자료**를 사용한다.
- SGIS 통계 기준연도는 2023년으로 통일한다.
- SGIS 집계구 공간경계는 2025년 2분기 자료를 사용한다.
- 지하철 운영 네트워크 기준일은 2023-12-31로 고정한다.
- 따라서 PROM 물리 현황과 SGIS·지하철 분석 사이에는 기준시점 차이가 있음을
  결과 해석과 보고서에 명시한다.

### 16.3 기존 경계 보관 및 비교

- 기존 B1·B2·B9·B10·C1·C2·M5·M6 중심 청라 국제업무단지 단독 경계와
  기존 처리 결과는 삭제하지 않고 다음 위치에 보관한다.


- 새 청라 업무중심지구 경계는 약 `1,036,788.38㎡`이며, 새 경계의 모든 지표는
  판교테크노밸리와 동일한 통합 스크립트와 산식으로 재산출한다.
- 판교와 새 청라의 통합 비교 결과는 다음 파일로 관리한다.

## 17. 시스템 구상 전 분석 완료 상태 (2026-06-13)

이 절의 내용은 문서 앞부분에 남아 있는 기존 청라 단독경계 및 과거 수치를
대체한다. 최종 비교 대상은 제1판교테크노밸리와 재정의한 청라 업무중심지구다.

### 17.1 필수 분석 완료

- PROM 용도지역별 면적 및 구성비
- 건축물 주용도별 연면적 및 구성비
- LUM, 평균 용적률, 미건축 필지 수·면적 비율
- SGIS 인구·가구·사업체·종사자 및 산업 대분류
- 직주비와 종사자 밀도
- OSM 도로망·교차로 밀도
- 핵심역 기준 30분·60분 접근권 인구·가구·종사자
- 0~60분 5분 단위 누적 접근성

청라 활성화 저조는 동일 산식 비교 결과로 검증한다. 청라는 미건축 필지 면적
비율이 약 91.3%이며, 판교보다 건축물 수·총연면적·종사자 밀도·직주비가 낮다.
